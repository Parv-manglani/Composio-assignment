How it was built

10 AI agents, working at once

No human touched the research by hand. 10 Claude agents split the 100 apps between them (10 each) and worked in parallel.

1

Split the list

10 categories → 10 agents, one category each.

2

Check Composio first

Each agent checked if Composio already had a toolkit for the app, before doing anything else.

3

Read the real docs

Each agent searched and read the app's actual developer docs — not guesses.

4

Fill in one form

Same fields for every app: auth, access, API, verdict, and a source link.

Where a human stepped in

  • Fact-checking. A second, independent pass re-read the primary docs for a sample of apps and fixed what the agents got wrong.
  • Judgment calls. E.g. does "free trial, but 3 seats minimum" count as self-serve? A human set one consistent rule and applied it everywhere.
  • Knowing when to stop. For a few tiny/unknown apps, a human confirmed the agents had genuinely searched enough before writing "no public docs found."

Code for both agents lives in agent/research.mjs and agent/verify.mjs — see the repo README to re-run them.

Bonus check: are the source links even real?

A script visited every source link for 25 random apps. Most of the "broken" ones turned out to be sites blocking bots (Otter.ai, PitchBook), not actually dead links — one was a genuinely missing Composio page, which matches what we'd already recorded.

What we actually got wrong

  • Reducto: said it had a working Composio toolkit — it doesn't yet. Fixed.
  • Twilio: mixed up two different MCP servers, one read-only and one that actually executes calls. Fixed.
  • GitHub: tool count was off by ~5%. Fixed to the exact number.
  • Pinterest Ads: one claim's source link broke before we could re-check it — marked unverified instead of guessing.
  • Paygent Connect, Waterfall.io, fanbasis: genuinely no public docs found for these — that's a real finding, not a failure.
